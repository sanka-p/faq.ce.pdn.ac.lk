You can develop on Kepler/Turing servers using VS Code's Remote SSH extension with the jump host configuration.

  1. Install the Remote - SSH extension in VS Code.
  2. Configure your local SSH config file (~/.ssh/config on Linux/Mac or C:\Users\YourUsername\.ssh\config on Windows): 
    Host aiken
    HostName aiken.ce.pdn.ac.lk
    User YOUR_USERNAME

Host tesla
    HostName tesla.ce.pdn.ac.lk
    User YOUR_USERNAME

Host kepler
    HostName 10.8.0.11
    User YOUR_USERNAME
    ProxyCommand ssh -W %h:%p aiken

Host turing
    HostName 10.8.0.14
    User YOUR_USERNAME
    ProxyCommand ssh -W %h:%p aiken
    Replace YOUR_USERNAME with your actual username (e.g., e16123).
  3. In VS Code, press F1 or Ctrl+Shift+P and type "Remote-SSH: Connect to Host"
  4. Select either kepler or turing from the list
  5. Once connected, you can install the Pylance extension on the remote server for Python code completion and select the correct conda environment for your project.

MobaXterm is a popular SSH client for Windows that provides both terminal access and file transfer capabilities.

  1. Download and install MobaXterm
  2. To connect to internal servers (Kepler/Turing), you need to use the jump host feature:
    • Click on "Session" → "SSH"
    • Remote host: Enter the internal IP (e.g., 10.8.0.11 for Kepler or 10.8.0.14 for Turing)
    • Username: Your username (e.g., e16123)
    • Go to the "Network settings" tab
    • Check "Connect through SSH gateway (jump host)"
    • Gateway SSH server: aiken.ce.pdn.ac.lk (or tesla.ce.pdn.ac.lk)
    • Gateway username: Your username (same as above)
    • Click OK to save and connect

Note: In this setup, Aiken/Tesla acts as the jump host (public IP server) to access Kepler/Turing (internal IP servers).

The choice of PyTorch or TensorFlow version depends on the CUDA version installed on each server. Using incompatible versions may result in GPU errors or degraded performance.

Recommended approach:

  1. Check the CUDA version on your target server: 
    nvidia-smi
    Look for the "CUDA Version" in the top right corner of the output.
  2. Refer to the compatibility guide for version recommendations:
    • PyTorch/TensorFlow versions for Peradeniya servers
    • PyTorch CUDA compatibility matrix
    • TensorFlow CUDA compatibility table
  3. Install using conda in your environment: 
    # Example for PyTorch with CUDA 11.8
conda install pytorch torchvision torchaudio pytorch-cuda=11.8 -c pytorch -c nvidia

# Example for TensorFlow with CUDA 11.8
conda install -c conda-forge cudatoolkit=11.8
pip install tensorflow==2.12.0

Tip: Always test your installation with a simple GPU test after installing:

# For PyTorch
python -c "import torch; print(torch.cuda.is_available())"

# For TensorFlow
python -c "import tensorflow as tf; print(tf.config.list_physical_devices('GPU'))"

Singularity containers allow you to package your entire development environment (including specific library versions, dependencies, etc.) and run it consistently across different servers.

When to use Singularity:

  • When you need a specific version of libraries that conflicts with system versions
  • When you want to ensure reproducibility across different servers
  • When you need root-like privileges for specific software installations
  • When you want to share your exact environment with collaborators

Internal IP servers (Kepler, Turing, Ampere, Ada) are not directly accessible from outside the university network. You have several options:

Option 1: Use a Jump Host (Recommended)

SSH to a public IP server first (Aiken or Tesla), then SSH to the internal server:

# First, connect to Aiken (public IP)
ssh your_username@aiken.ce.pdn.ac.lk

# Then from Aiken, connect to the internal server
ssh your_username@kepler.ce.pdn.ac.lk
# OR
ssh your_username@turing.ce.pdn.ac.lk
# OR
ssh your_username@ampere.ce.pdn.ac.lk

Option 2: Direct SSH with ProxyJump (Single Command)

Configure your ~/.ssh/config file as shown in the VS Code Remote SSH section above, then connect directly:

ssh kepler   # Uses aiken as jump host automatically

Option 3: Connect from Peradeniya Network

If you're on a computer within the university network, you can connect directly:

ssh your_username@kepler.ce.pdn.ac.lk
